# Validation

The implementation and reports must measure:

| Metric | Purpose |
| --- | --- |
| Valid decodes by threshold method | Prioritize useful threshold methods. |
| False positives or empty decodes by threshold method | Identify risky texture generators. |
| Finder triples without decodes by threshold method | Identify work-reduction targets. |
| Positives rescued by inverted polarity | Quantify polarity value. |
| High-realism but bad decode outcomes by threshold method | Improve realism scoring. |
